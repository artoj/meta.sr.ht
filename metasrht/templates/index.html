git.sr.ht

Our git repository hosting service is git.sr.ht. Features include:

  • Public, private, and "unlisted" repositories
  • Fine grained access control, including access for users without accounts
  • Software release management
  • Integration with builds.sr.ht and bugs.sr.ht

builds.sr.ht

builds.sr.ht is our continuous integration service. Features include:

  • Builds on various Linux distros and BSDs
  • Custom build images
  • Run anonymous jobs that aren't tied to a repository
  • Post-build triggers for email, webhooks, etc
  • Powerful API for complex build scenarios

meta.sr.ht

Account management is done via meta.sr.ht. Features include:

  • Extended OAuth features for authorization to other sr.ht and third party services
  • Security features like two-factor authentication and detailed audit logs
  • PGP encrypted and signed emails

Under development

lists.sr.ht

Powerful mailing list software. Planned features include:

  • Customizable patch review queues
  • Mail submission and patch review via the web
  • Tools for working with third party mailing lists
  • Integration with builds.sr.ht and git.sr.ht

bugs.sr.ht

bugs.sr.ht provides bug tracking and release automation. Planned features include:

  • Tests patches with builds.sr.ht
  • Private bug reports and bug trackers for security issues
  • Project release planning
  • Import and export to and from other bug trackers

files.sr.ht

File hosting for releases and whatever else you need. Planned features include:

  • Public and private uploads
  • Direct, unmolested links to your files
  • Code snippet support
  • HTTPS and FTP access
